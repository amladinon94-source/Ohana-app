02 · Point, don't describe

Drop a pin where something's off

You comment on the real element, like in Figma. The agent reads the pin, knows what you mean, and replies.

  1. 1Turn on comment mode (⇧⌘M) and click any element on this page.
  2. 2Type what to change. A numbered pin stays anchored to the element.
  3. 3In the Comments panel (⇧⌘F), the agent replies and resolves.

05 · From idea to repository

A single thread: from flow to real code

Nothing stays a drawing. You define in Flows, iterate the prototype with comments, and once it's ready: "Generate handoff" distills the project into Linear-ready tickets, and "Ship to repository" implements it with the real components of your design system.
